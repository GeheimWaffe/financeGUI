# Principes de construction d'une IHM de type page avec streamlit

Le code source d'une page est stocké dans un module.

## Partie I
La première partie du module contient les imports.

## Partie II
on déclare les clés des widgets qui doivent hériter de filtres utilisateurs communs à toutes les pages.

## Partie III
Interactivité : fonctions qui déclenchent un changement de comportement de la page

## Partie IV
Déclaration des call backs : on écrit les fonctions qui changent le comportement de la page

## Partie V
Construction de l'IHM : Une fonction principale porte le code de la page.

### Initialisation des réglages par défaut
Au début de la fonction, on initialise les widgets héritant des filtres utilisateurs, avec le contenu des filtres utilisateurs
